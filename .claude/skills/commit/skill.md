# Commit Skill

Review git status and create a commit with an appropriate message.

## Steps

1. Run `git status` and `git diff` to understand what changed
2. Stage all changes with `git add -A`
3. Write a concise commit message based on the actual changes
4. Commit

## Rules

- Commit message should be short (under 72 chars), imperative tense, lowercase
- Do not include Co-Authored-By or any trailer lines
- No confirmation needed — just commit

## Trigger

User invokes `/commit`
